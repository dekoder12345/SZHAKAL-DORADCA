## 1. Metadata generation became a first-class feature

Nikodem requested that every pipeline should generate, at the end of the run:

- a social media description with paragraphs
- up to 5 hashtags inside that SM description
- a YouTube SEO title
- YouTube tags as comma-separated tags (NOT hashtags)

This was explicitly framed as:

- bonus content in the SM description should give viewers a reason to read from line 1
- bonus must stay truthful and related to the video
- YouTube tags are separate from SM hashtags

Resulting architecture in the workspace:

- metadata generation exists across all 4 repos
- sibling repos use `metadata_gen.py`
- `Scenariusze` has metadata generation wired as **Faza 4**
- `Scenariusze` also has standalone `generate_metadata.py`
- pipelines expose `--skip-metadata` / `--only-metadata`

Durable takeaway:

- metadata is now part of the intended end-state of generated videos
- it is no longer just an idea from chat

## 2. New "Whisper pipeline" idea was intentionally NOT made into a 5th repo

Nikodem asked for a new pipeline that would:

- scan project folders for `.txt` files to gather context
- transcribe video/audio with OpenAI speech-to-text
- extract context from images
- build Polish context from those materials before scenario generation

Codex recommendation:

- do **NOT** create a 5th pipeline/repo
- implement this as **ETAP 0 — ingest kontekstu** inside `Scenariusze`

Important architectural decision:

- text/audio/video/image ingestion belongs before scenario generation in `Scenariusze`
- it should be cached and resumable
- it should enrich the scenario prompt, not replace the rest of the pipeline
